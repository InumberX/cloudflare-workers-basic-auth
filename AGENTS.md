# Repository Guidelines

## Project Structure & Module Organization

This repository is a small TypeScript package for Cloudflare Workers Basic Auth.

- `src/` contains publishable library code. Public exports start at `src/index.ts`.
- `src/basic-auth/` holds credential decoding, verification, response helpers, and timing-safe comparison logic.
- `src/worker-fetch/` contains the Worker integration wrapper.
- `tests/` contains Vitest suites named `*.test.ts`.
- `dist/` is generated by the build and should not be edited manually.

## Build, Test, and Development Commands

Use Node.js `>=20.19.0`.

- `npm install` installs dependencies.
- `npm run typecheck` runs TypeScript without emitting files.
- `npm test` starts Vitest in watch mode for local development.
- `npm test -- --run` runs the test suite once, matching CI behavior.
- `npm run lint` checks `src/` and `tests/` with Oxlint.
- `npm run format` verifies formatting with Oxfmt.
- `npm run build` cleans and compiles the package into `dist/`.
- `npm run pre-commit` type-checks, then applies lint and formatting fixes.

Before opening a PR, run the CI sequence: `npm run typecheck && npm run lint && npm run format && npm test -- --run && npm run build`.

## Coding Style & Naming Conventions

The codebase uses ES modules and TypeScript. Follow Oxfmt settings: 2-space indentation, no semicolons, single quotes, trailing commas where valid, and a 120-character line width. Keep imports grouped as configured in `.oxfmtrc.json`.

Use descriptive camelCase for functions and variables (`verifyBasicAuth`), PascalCase for exported types (`VerifyBasicAuthOptions`), and kebab-case filenames (`timing-safe-equal.ts`). Prefer small focused modules and preserve the library’s Web Standards-only design.

## Testing Guidelines

Tests use Vitest and live in `tests/`. Name suites `feature-name.test.ts` and cover success and failure paths, especially malformed credentials, UTF-8 handling, response headers, and method-routing edge cases. No coverage threshold is configured; add regression tests for every behavior change.

## Commit & Pull Request Guidelines

Recent history uses short prefixed commit messages such as `fix: update vitest` and `add: cloudflare workers basic auth (#1)`. Keep subjects imperative and scoped.

Pull requests should include a concise behavior summary, linked issues when applicable, tests added or updated, and any security or Cloudflare Workers compatibility implications.

## Security & Configuration Tips

Do not commit secrets. Worker credentials should be configured as `BASIC_AUTH_USER` and `BASIC_AUTH_PASS` secrets. Changes to authentication behavior should preserve fail-closed handling, constant-time comparison, and safe-method asset routing unless the tradeoff is explicitly documented.
